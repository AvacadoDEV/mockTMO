# Project North Star, a mock TMO case study

A portfolio piece showing what a Transformation Management Office puts in front of leadership in a working week. Charter, executive SteerCo deck, portfolio dashboard, risk log, weekly decisions memo, and value tracking model.

Built around a fictional Canadian omnichannel retailer ("Maple Retail Co.") fourteen months into a 36-month, five-workstream transformation program.

Live at **[northstar.anuprao.dev](https://northstar.anuprao.dev)**.

## What's in here

Everything lives in `index.html`. One file, no build step, no backend, no dependencies. Double-click to open it, or serve the folder with anything (`python -m http.server`, `npx serve`, etc.).

Downloadable artifacts sit in `downloads/`:

- `project-north-star-charter.docx`. Initiative charter (Word).
- `project-north-star-steerco.pptx`. Ten-slide SteerCo deck (PowerPoint).
- `project-north-star-tracker.xlsx`. Portfolio dashboard, risk log, value tracking model (Excel, three tabs).
- `project-north-star-wed-preread.docx`. Weekly Exec Decisions pre-read (Word).

## Deployment

Hosted on GitHub Pages from `main`. The `CNAME` file points the site at `northstar.anuprao.dev`; the DNS record lives on the `anuprao.dev` zone as a CNAME to `anuprao.github.io`.

## Why fictional?

Every artifact here would look substantially the same at any $5&ndash;20B retailer partway through a transformation. Naming a real company would trade signal for the impression of endorsement.

Not affiliated with any retailer named or unnamed.
